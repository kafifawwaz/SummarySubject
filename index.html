Operating Systems

Perantara antara hardware dan pengguna. OS mengatur semua sumber daya komputer agar bisa digunakan secara efisien.

🏠 Analogi
OS itu seperti manajer hotel. Tamu (aplikasi) datang minta kamar (RAM), makan (CPU), listrik (I/O). Manajer (OS) mengatur siapa dapat apa, kapan, dan berapa lama — tanpa tamu perlu tahu cara kerja kelistrikan hotelnya.
Konsep Utama
  • Apa itu OS? Program yang jadi perantara user ↔ hardware. Tujuan: eksekusi program, mudah dipakai, efisien gunakan hardware.
  • 4 Komponen Sistem Komputer: Hardware → OS → Application Programs → Users.
  • OS sebagai Resource Allocator: Mengatur semua resource dan menentukan siapa yang dapat resource ketika ada konflik.
  • Kernel: Satu-satunya program yang berjalan terus-menerus di komputer. Semua yang lain adalah system/application program.
Boot Process & Interrupt
Power ON → Bootstrap (ROM/EPROM) → Load Kernel → OS Running

Interrupt = sinyal dari hardware/software yang "mengganggu" CPU agar ditangani. OS adalah interrupt-driven — ia hanya bekerja saat ada yang minta perhatian. Trap/Exception = interrupt dari software (contoh: divide by zero).

Memory & Storage Hierarchy
⚡

Registers (Tercepat)

Di dalam CPU. Kapasitas kecil banget, tapi akses dalam nanosecond. Volatile (hilang saat mati).

🗄️

Cache

Penyimpanan perantara yang menyimpan salinan data dari main memory. Prinsip: cek cache dulu sebelum ke main memory.

💾

Main Memory (RAM)

Satu-satunya storage yang bisa diakses CPU secara langsung. Random access. Volatile.

💿

Secondary Storage (HDD/SSD)

Besar, non-volatile, lebih lambat. SSD lebih cepat dari HDD tapi lebih mahal.

Urutan dari terlambat ke tercepat: Magnetic Tape → Optical Disk → HDD → Nonvolatile Memory (SSD) → Main Memory → Cache → Registers.

Process & Memory Management
  • Process vs Program: Program = pasif (file di disk). Process = aktif (program yang sedang dieksekusi). Process butuh CPU, memory, I/O, files.
  • Multiprogramming: Beberapa job di memory sekaligus. Saat satu job tunggu I/O, CPU switch ke job lain. CPU tidak pernah nganggur.
  • Timesharing/Multitasking: CPU switch antar job sangat cepat sehingga user merasa interaktif. Response time < 1 detik.
  • Dual-Mode Operation: User mode vs Kernel mode. Mode bit di hardware. Instruksi privileged hanya di kernel mode. Proteksi OS dari program nakal.
  • Virtual Memory: Proses bisa berjalan walau tidak semua bagiannya ada di RAM. Sisanya di swap space di disk.
Computer Architecture
🖥️

Symmetric Multiprocessing (SMP)

Semua CPU setara, menjalankan semua task. Lebih umum dipakai. Throughput meningkat seiring jumlah CPU.

🔗

Clustered Systems

Beberapa sistem bekerja bersama via SAN (Storage Area Network). Untuk high availability — kalau satu mati, yang lain lanjutkan.

🎮

Virtualization

OS bisa jalankan OS lain di dalamnya (VM). VMM (Virtual Machine Manager) seperti VMware, VirtualBox. Dasar dari Cloud Computing.

⚡ Poin Kunci untuk Ujian

  1. OS = resource allocator + control program. Kernel = satu program yang always running.
  2. Boot: Bootstrap Program (ROM) → Load Kernel. OS adalah interrupt-driven.
  3. Hirarki storage: Registers paling cepat, Magnetic Tape paling lambat.
  4. Multiprogramming = CPU selalu sibuk. Timesharing = user merasa interaktif.
  5. Dual-mode (user/kernel mode) melindungi OS dari error/abuse program.
  6. Virtualization = dasar Cloud Computing.

Security in Computer Networks

Bagaimana komunikasi digital dijaga dari mata-mata, manipulasi, dan serangan.

📮 Analogi
Network security seperti pengiriman surat rahasia. Alice = pengirim. Bob = penerima. Trudy = penyusup yang bisa mencuri, memalsukan, atau mengubah surat. Kita butuh: amplop yang tidak bisa dibuka (enkripsi), stempel asli (autentikasi), dan segel anti-rusak (message integrity).
4 Pilar Network Security
🔒

Confidentiality

Hanya pengirim & penerima yang paham isi pesan. Dicapai dengan enkripsi. Trudy bisa lihat ciphertext tapi tidak bisa baca.

✅

Authentication

Pengirim dan penerima bisa konfirmasi identitas satu sama lain. "Kamu benar-benar Alice?"

🛡️

Message Integrity

Pesan tidak berubah saat transit. Pakai digital signature / hash function.

🌐

Availability

Layanan tetap bisa diakses. Mencegah DoS (Denial of Service) attack.

Kriptografi
🔑 Analogi Symmetric vs Asymmetric
Symmetric = satu kunci gembok yang sama antara Alice dan Bob (tapi gimana cara sharingnya?). Asymmetric (Public Key) = Bob pasang kotak surat terbuka (public key) di depan rumah. Siapa saja bisa masukin surat. Tapi cuma Bob yang punya kuncinya untuk buka kotak itu (private key).
EnkripsiKunciKecepatanContoh
SymmetricSama untuk enkripsi & dekripsiCepat (100x RSA)DES, 3DES, AES
Asymmetric (Public Key)Public key (enkripsi), Private key (dekripsi)LambatRSA, Diffie-Hellman
  • DES (Data Encryption Standard): 56-bit key. Bisa di-brute force dalam sehari → tidak aman.
  • 3DES: DES dijalankan 3x dengan 3 key berbeda. Lebih aman.
  • AES (Advanced Encryption Standard): 128-bit blocks, key 128/192/256 bit. Brute force butuh 149 TRILIUN tahun.
  • RSA: Pakai bilangan prima besar. Public key = (n, e). Private key = (n, d). Keamanan berbasis sulitnya faktorisasi bilangan besar.
Authentication Protocols (ap1.0 → ap5.0)
  • ap1.0: Alice bilang "I am Alice" → Gagal: Trudy bisa claim hal yang sama.
  • ap2.0: Alice kirim IP address-nya → Gagal: IP spoofing bisa dilakukan Trudy.
  • ap3.0: Alice kirim password → Gagal: Playback attack — Trudy record dan replay.
  • ap4.0: Bob kirim nonce R. Alice enkripsi R dengan shared key → Bob tahu Alice "live". Tapi butuh shared key duluan.
  • ap5.0: Pakai public key + nonce. Masih ada flaw: Man-in-the-Middle attack oleh Trudy.
  • Solusi: Certification Authority (CA) yang bind public key ke identitas nyata. CA menandatangani sertifikat secara digital.
Digital Signatures & Hash Functions
  • Digital Signature: Bob enkripsi pesan dengan private key-nya → hanya Bob yang bisa buat tanda tangan itu. Non-repudiation: Bob tidak bisa menyangkal.
  • Hash Function: Konversi pesan besar → fixed-size "fingerprint" (H(m)). Satu arah — tidak bisa di-reverse. MD5 = 128-bit, SHA-1 = 160-bit.
  • Praktis: Sign hash(m) bukan m langsung → lebih efisien. Disebut "message digest".
TLS & IPsec
🔐

TLS (Transport Layer Security)

Protokol di atas TCP. Dipakai HTTPS (port 443). Gabungan: confidentiality (enkripsi symmetric), integrity (hash), authentication (public key). TLS 1.3 pakai Diffie-Hellman untuk key exchange.

🛡️

IPsec

Enkripsi di level Network layer. Dua mode: Transport (enkripsi payload saja) & Tunnel (enkripsi seluruh datagram). Protokol: AH (integrity, no confidentiality) & ESP (lebih lengkap, termasuk confidentiality).

Firewalls & IDS
🏰 Analogi
Firewall = Satpam di pintu masuk gedung yang cek ID kartu. IDS (Intrusion Detection System) = CCTV di dalam gedung yang memantau kalau ada yang berperilaku mencurigakan meski sudah masuk.
  • Stateless Packet Filter: Cek header tiap paket (IP, port, protocol) satu per satu. Tidak ingat koneksi sebelumnya. Bisa ditipu paket "nonsensical".
  • Stateful Packet Filter: Ingat status koneksi TCP. Lebih cerdas — paket ACK tanpa koneksi sebelumnya = mencurigakan.
  • Application Gateway: Filter di level aplikasi. Contoh: proxy server untuk Telnet, HTTP.
  • IDS: Deep packet inspection — cek isi paket, bukan cuma header. Deteksi port scanning, DoS, dll.

⚡ Poin Kunci untuk Ujian

  1. 4 pilar: Confidentiality, Authentication, Integrity, Availability.
  2. Symmetric = cepat tapi masalah key sharing. Asymmetric = lambat tapi aman untuk key exchange.
  3. AES jauh lebih aman dari DES. RSA pakai bilangan prima besar.
  4. Praktis: pakai RSA untuk tukar session key, lalu pakai AES untuk data.
  5. Digital signature = enkripsi hash(m) dengan private key. Non-repudiation.
  6. CA mengatasi masalah Man-in-the-Middle di ap5.0.
  7. TLS = security di atas TCP. IPsec = security di network layer.
  8. Firewall vs IDS: filter di pintu vs deteksi di dalam.

Wireless and Mobile Networks

Elemen, protokol, dan tantangan jaringan nirkabel — dari WiFi sampai 5G.

📡 Analogi
Wireless network seperti radio FM. Ada stasiun pemancar (base station/AP), dan ada pendengar (wireless host). Tapi bedanya — di wireless network, pendengar juga bisa kirim sinyal balik, dan banyak "pendengar" yang harus berbagi frekuensi yang sama tanpa tabrakan.
Elemen Jaringan Wireless
  • Wireless Hosts: Laptop, smartphone, IoT. Bisa bergerak (mobile) atau diam. Wireless ≠ mobile!
  • Base Station: Terhubung ke wired network. Relay antara wired network & wireless hosts. Contoh: cell tower, 802.11 AP.
  • Wireless Link: Menghubungkan host ke base station. Ada koordinasi multiple access.
  • Infrastructure Mode: Host terhubung via base station → handoff saat pindah BS. Ad hoc mode: Tidak ada base station, peer-to-peer.
Tantangan Wireless (vs Wired)
📉

Decreased Signal Strength

Sinyal radio melemah saat melewati materi (path loss). Makin jauh, makin lemah.

📶

Interference

Frekuensi 2.4 GHz dipakai banyak perangkat (WiFi, microwave, motor). Saling gangu satu sama lain.

🔁

Multipath Propagation

Sinyal radio memantul dari objek/tanah → sampai di tujuan pada waktu sedikit berbeda (ghosting).

👻

Hidden Terminal Problem

A dan C tidak bisa saling dengar tapi keduanya kirim ke B sekaligus → collision di B tapi A & C tidak tahu!

IEEE 802.11 WiFi
  • CSMA/CA (Collision Avoidance): Berbeda dari wired yang pakai CSMA/CD. Wireless pakai CA karena collision tidak bisa dideteksi. Sebelum kirim, tunggu DIFS, lalu random backoff.
  • RTS/CTS: Sender kirim Request-to-Send kecil → AP balas Clear-to-Send → semua node lain tahu ada yang mau kirim, diam dulu. Solusi hidden terminal problem.
  • Association: Host scan channel, dengar beacon frames (berisi SSID & MAC AP), pilih AP, autentikasi, dapat IP via DHCP.
  • Rate Adaptation: AP & mobile dinamis ubah transmission rate sesuai SNR (Signal-to-Noise Ratio). Jauh dari AP → SNR rendah → pakai rate lebih rendah.

4G LTE Architecture
📱

UE (User Equipment)

Perangkat dengan SIM card yang menyimpan IMSI (64-bit identity). Smartphone, tablet, IoT.

📡

eNode-B (Base Station)

Kelola radio resources di area cakupannya. Koordinasi handover dengan BS tetangga. Lebih aktif dari WiFi AP.

🔑

MME (Mobility Management Entity)

Otentikasi device, kelola handover antar sel, setup tunnel untuk data path.

🏠

HSS (Home Subscriber Service)

Database identitas & layanan pelanggan. Authenticator utama — tahu siapa kamu dan apa layanan yang kamu bayar.

Mobility: Indirect vs Direct Routing
🏠 Analogi Indirect Routing
Indirect routing seperti kirim surat ke teman yang sedang merantau melalui rumah orang tuanya. Surat selalu ke alamat rumah orang tua dulu, baru diteruskan ke lokasi teman sebenarnya. Tidak efisien, tapi transparan — pengirim tidak perlu tahu si teman lagi di mana.
  • Indirect Routing: Data ke home network → diteruskan ke visited network. Transparan bagi pengirim tapi ada "triangle routing" yang tidak efisien.
  • Direct Routing: Pengirim tanya home HSS di mana lokasi mobile, lalu kirim langsung. Lebih efisien tapi tidak transparan.
  • Handover (Handoff): Mobile pindah dari satu BS ke BS lain. Source BS kasih tau target BS, target BS pre-alokasi slot, lalu MME update tunnel endpoint ke target BS.

⚡ Poin Kunci untuk Ujian

  1. Wireless ≠ mobile. Wireless host bisa statis, mobile bisa pakai kabel.
  2. Tantangan utama: path loss, interference, multipath, hidden terminal.
  3. 802.11 pakai CSMA/CA (bukan CD). RTS/CTS solusi hidden terminal.
  4. 4G: UE → eNode-B → S-GW → P-GW → Internet. MME untuk kontrol, HSS untuk identitas.
  5. Indirect routing: transparan tapi "triangle routing". Direct routing: efisien tapi tidak transparan.
  6. 5G lebih cepat, latensi rendah, enkripsi IMSI, keputusan auth di home network.

Cloud Computing

Model komputasi yang menyediakan resource IT sesuai kebutuhan melalui internet — bayar sesuai pakai.

💡 Analogi
Cloud computing seperti PLN (listrik). Kamu tidak perlu beli generator sendiri, tidak perlu perbaiki jika rusak, dan kamu bayar sesuai yang kamu pakai (kwh). John McCarthy sudah prediksi ini di tahun 1961: "computing may someday be organized as a public utility." Dan terbukti!
Sejarah Singkat
1961
John McCarthy prediksi computing sebagai public utility seperti telepon.
1969
Leonard Kleinrock (ARPANET): jaringan komputer akan jadi infrastruktur seperti utilitas publik.
1999
Salesforce.com: pionir Software as a Service (SaaS) untuk enterprise.
2006
Amazon Web Services (AWS) launch — storage, compute, business functions sebagai layanan.
Definisi NIST (yang paling penting!)
📖 Definisi
Cloud = model untuk akses jaringan ubiquitous, convenient, on-demand ke shared pool of configurable computing resources (networks, servers, storage, apps, services) yang bisa rapidly provisioned and released dengan minimal management effort.

Terdiri dari: 5 karakteristik + 3 service models + 4 deployment models.
5 Karakteristik Cloud
🎯

On-Demand Usage

Self-service. Tidak perlu telepon provider. Langsung provision sendiri, otomatis berjalan.

🌍

Ubiquitous Access

Bisa diakses dari mana saja, kapan saja, perangkat apa saja (laptop, HP, tablet).

👥

Multitenancy

Satu instance resource melayani banyak consumer (tenant) yang terisolasi satu sama lain. Efisien via resource pooling.

📈

Elasticity

Scale up/down secara otomatis sesuai demand runtime. Bayar saat butuh, lepas saat tidak butuh.

📊

Measured Usage

Pakai berapa, bayar sebanyak itu (pay-as-you-go). Semua terukur dan terpantau.

  • Resiliency (bonus): Redundant IT resources di lokasi berbeda. Kalau satu cloud down, failover otomatis ke cloud lain.
  • Horizontal Scaling (Scale Out/In): Tambah/kurangi jumlah instance yang identik. Lebih umum di cloud.
  • Vertical Scaling (Scale Up/Down): Ganti instance dengan yang lebih kuat/lemah. Butuh downtime, kurang umum.
3 Service Models (Delivery Models)
🍕 Analogi Pizza
IaaS = kamu beli bahan-bahan pizza sendiri dan masak sendiri di dapur sewa.
PaaS = dapur sudah tersedia lengkap dengan oven dan alat. Kamu tinggal bawa bahan dan masak.
SaaS = pizza sudah jadi, tinggal makan. Tidak perlu tahu resepnya.
ModelApa yang Dikelola ProviderApa yang Dikelola UserContoh
IaaSHardware, networking, virtualizationOS, runtime, apps, dataAWS EC2, Google Compute Engine
PaaSInfrastructure + OS + runtime + middlewareApplications + dataGoogle App Engine, Heroku
SaaSSemua! Termasuk aplikasinyaCuma konfigurasi & data userGmail, Google Docs, Salesforce
4 Deployment Models
🌐

Public Cloud

Milik provider, diakses publik via internet. Bayar atau monetisasi iklan. Contoh: AWS, Azure, GCP.

🔒

Private Cloud

Dimiliki & digunakan satu organisasi saja. Lebih aman dan control penuh, tapi lebih mahal.

🔀

Hybrid Cloud

Gabungan public + private. Data sensitif di private, yang tidak sensitif di public. Kompleks untuk dikelola.

🌐🌐

Multicloud

Pakai layanan dari beberapa cloud provider berbeda. Hindari vendor lock-in, gunakan best-of-breed services.

Risiko dan Tantangan Cloud
  • Overlapping Trust Boundaries: Tanggung jawab keamanan data dibagi antara cloud provider dan consumer. Rawan celah di "sambungan" keduanya.
  • Shared Security Responsibility: Data milik consumer, tapi infrastruktur milik provider. Siapa bertanggung jawab jika terjadi breach?
  • Limited Portability: Pindah dari satu cloud provider ke lain susah karena beda teknologi proprietary. Vendor lock-in.
  • Multiregional Compliance: Data center bisa di negara lain. Ada regulasi soal data harus disimpan di negara tertentu.
  • Cost Overruns: Tanpa governance yang baik, biaya cloud bisa meledak. Pay-as-you-go bisa jadi jebakan kalau tidak dipantau.

⚡ Poin Kunci untuk Ujian

  1. 5 karakteristik NIST: On-demand, Ubiquitous, Multitenancy, Elasticity, Measured Usage.
  2. IaaS = infrastruktur saja. PaaS = platform siap pakai. SaaS = software siap pakai.
  3. 4 deployment: Public, Private, Hybrid, Multicloud.
  4. Horizontal scaling (out/in) lebih umum di cloud vs Vertical scaling (up/down).
  5. Cloud provider ≠ bertanggung jawab penuh atas keamanan data. Shared responsibility.
  6. Resiliency = redundant resource di berbagai lokasi untuk failover otomatis.

Internet of Things (IoT)

Jaringan objek fisik yang saling terhubung dan bisa "berkomunikasi" lewat internet.

🌍 Analogi
IoT itu seperti dunia yang punya "syaraf". Setiap benda (kulkas, lampu, motor, jantung pasien) adalah sensor yang merasakan dunia fisik. Otak (cloud/AI) mengolah datanya. Dan efektornya (actuator, alarm, pompa) mengambil tindakan. Dunia fisik dan digital jadi satu kesatuan yang hidup.
Sejarah & Definisi
  • 1982: Mesin Coca-Cola di Carnegie Mellon — device pertama yang terhubung internet! Bisa lapor persediaan dan suhu minuman secara remote.
  • 1991: Mark Weiser memperkenalkan konsep "ubiquitous computing" — komputer menyatu dengan lingkungan.
  • 1999: Kevin Ashton menciptakan istilah "Internet of Things" saat bekerja dengan RFID di MIT.
  • 2002: Forbes pertama kali gunakan istilah "Internet of Things" secara resmi.
IoT vs Teknologi Terkait
TeknologiPerbedaan utama dengan IoT
M2M (Machine-to-Machine)Komunikasi point-to-point, biasanya pakai jaringan cellular tertutup. IoT lebih luas, pakai internet terbuka.
WSN (Wireless Sensor Network)Fokus ke sensor, belum tentu terhubung ke internet. IoT menghubungkan ke layanan cloud dan analytics.
CPS (Cyber-Physical System)Lebih ke integrasi komputasi dengan proses fisik. IoT lebih ke connectivity dan identification.
Embedded SystemSistem komputer dalam perangkat fisik, belum tentu terhubung internet. IoT pasti terhubung.
5 Karakteristik Utama IoT
📡

Sensor Data Acquisition

Sensor kumpulkan data lingkungan dan kirim ke cloud untuk storage & analytics.

🔗

Connectivity

Interkoneksi fisik dan virtual via infrastruktur komunikasi global (wired & wireless).

🧠

Device Heterogeneity & Intelligence

Device dari hardware berbeda-beda bisa interoperate. Ada ambient intelligence di hardware/software level.

📈

Scalability

Miliaran device bisa terhubung. Interaksi bergeser dari human-to-device ke device-to-device.

🔒

Security

Keamanan harus ada di level jaringan DAN di perangkat akhir (end-device).

Enabling Technologies IoT
  • Identification: Setiap "thing" punya identitas unik. Metode: UCode, EPC, QR Code, UPC. IPv6 untuk global addressing (128-bit = cukup untuk semua atom di bumi, hampir).
  • Sensors & Actuators: Sensor = ubah input fisik → data digital (suhu, cahaya, tekanan). Actuator = ubah sinyal listrik → aksi fisik (motor, pompa, lampu).
  • RFID (Radio Frequency Identification): Identifikasi objek tanpa kontak fisik via gelombang radio. Tag (di objek) ↔ Reader. Dua tipe: Active tag (punya baterai, jangkauan jauh) & Passive tag (tenaga dari sinyal reader, murah).
  • WSN (Wireless Sensor Network): Jaringan sensor yang kumpulkan & kirim data ke tujuan (sink/cloud). Pakai IEEE 802.15.4 untuk low-power communication.
  • Connectivity: WiFi, Bluetooth, ZigBee, LTE untuk wireless. USB, Ethernet untuk wired. Pilih berdasarkan range, data rate, power consumption.
IoT Information Value Loop
Data Creation (Sensor) → Communication → Aggregation → Analysis (AI) → Action/Goal
IoT Architecture (3-Layer)
  • Perception Layer (bawah): Sensor & perangkat fisik yang kumpulkan data dari lingkungan.
  • Network Layer (tengah): Transmisi data dari perangkat ke cloud/data center.
  • Application Layer (atas): Proses data dan komunikasi dengan end-user. Smart home, healthcare, dll.
Augmented Intelligence vs Augmented Behavior
🤖

Augmented Intelligence

AI yang menganalisis data IoT. Tiga jenis: Descriptive (apa yang terjadi?), Predictive (apa yang akan terjadi?), Prescriptive (apa yang harus dilakukan?).

⚙️

Augmented Behavior

Tindakan nyata yang diambil berdasarkan analisis. M2M (mesin ke mesin otomatis) & M2H (mesin ke manusia). Dari data science ke behavioral science.

⚡ Poin Kunci untuk Ujian

  1. Kevin Ashton coined "IoT" di 1999. Mulai dari Coke machine yang terhubung internet di 1982.
  2. IoT ≠ M2M ≠ WSN ≠ CPS. Tapi IoT mengintegrasikan konsep dari semua teknologi tersebut.
  3. 5 karakteristik: Sensor, Connectivity, Heterogeneity, Scalability, Security.
  4. RFID: Active tag (punya baterai, jauh) vs Passive tag (tenaga dari reader, murah).
  5. 3-layer arch: Perception → Network → Application.
  6. Augmented Intelligence = analisis data. Augmented Behavior = tindakan nyata.

Blockchain

Distributed ledger yang terdesentralisasi, immutable, dan aman secara kriptografi.

📒 Analogi
Blockchain seperti buku rekening publik yang ribuan orang pegang salinannya. Kalau kamu mau catat transaksi baru, semua orang harus setuju (konsensus). Dan karena setiap halaman (block) nyambung ke halaman sebelumnya dengan "lem" kriptografi (hash), tidak bisa ada yang diam-diam merobek atau mengganti halaman lama tanpa semua orang tahu.
Dari Ledger ke Blockchain
  • Ledger Tradisional: Sistem pencatatan transaksi. Ada sejak era Babilonia. Butuh kepercayaan pada pihak ketiga (bank, pemerintah).
  • DLT (Distributed Ledger Technology): Ledger yang bisa diakses, diverifikasi, dan diupdate secara simultan oleh banyak pihak dalam jaringan. Triple-entry accounting: debit + kredit + catatan di immutable ledger.
  • Blockchain: Bentuk DLT paling populer. Transaksi disimpan dalam block-block yang saling terhubung secara kriptografis. Terdesentralisasi — tidak ada satu otoritas pun yang mengontrol.
Sejarah Singkat
1991
Stuart Haber & W. Scott Stornetta memperkenalkan konsep blockchain untuk timestamp dokumen.
2008
Satoshi Nakamoto mempopulerkan blockchain untuk Bitcoin sebagai public transaction ledger.
Anatomi Block

Genesis Block = block pertama, tidak punya parent block. "Awal dari segalanya."

Hash Function & Enkripsi
  • Hash Function: Input → output unik (fixed-length string). Satu arah (tidak bisa di-reverse). Avalanche effect: perubahan kecil input → perubahan total output. Dipakai di setiap block untuk integritas.
  • Simple Hash: Brute force. Untuk dokumen dengan ukuran tetap (block header).
  • Merkle Tree Hash: Data diatur sebagai binary tree, hash dilakukan berpasangan. Efisien untuk data besar (O(log n)). Dipakai untuk transaction integrity di block body.
  • ECC (Elliptic Curve Cryptography): Preferred di blockchain (vs RSA). 256-bit ECC = security setara 3072-bit RSA. Lebih efisien.
  • Decentralized Identifier (DID): Identitas pseudo-anonim yang diamankan private key. Satu orang bisa punya banyak DID. Terkait dengan verifiable credentials.
Ekosistem Blockchain
💻

Nodes

Komputer yang menyimpan salinan ledger. Memverifikasi & meneruskan transaksi. Transaksi baru harus disetujui mayoritas node.

⛏️

Miners

Node khusus yang buat block baru dengan menyelesaikan puzzle kriptografi (cari nonce). Mendapat reward digital sebagai insentif.

👛

Wallets

Kelola private & public keys. Deterministic wallet: bisa recover dari seed phrase. Non-deterministic: tidak ada recovery jika private key hilang.

📜

Smart Contracts

Program yang otomatis eksekusi terms & conditions saat kondisi terpenuhi. Contoh: transfer otomatis saat barang diterima.

Konsensus Mechanism
🏰 Analogi Byzantine Generals Problem
Beberapa jenderal harus sepakat untuk serang benteng secara serentak. Tapi komunikasi bisa dicegat musuh. Bagaimana mereka yakin semua jenderal terima pesan yang sama? Blockchain menyelesaikan ini dengan Proof-of-Work: jika semua peserta mengerjakan puzzle matematika, solusi pertama yang valid menandakan pesan sudah diterima semua orang.
  • Proof-of-Work (PoW): Miner berlomba cari nonce. Pertama yang solve dapat reward. Butuh banyak listrik tapi aman (51% attack butuh hardware luarbiasa mahal).
  • Proof-of-Stake (PoS): Validator dipilih berdasarkan jumlah cryptocurrency yang mereka "stake" (jaminkan). Lebih hemat energi dari PoW.
  • Delegated PoS: Token holders pilih "delegates" yang lakukan validasi. Lebih skalabel.
Evolusi Blockchain
  • Blockchain 1.0: Digital currency, cryptocurrency, gambling. Bitcoin.
  • Blockchain 2.0: Smart contracts, dApps, DAO (Decentralized Autonomous Organization). Ethereum.
  • Blockchain 3.0: Meluas ke education, health, science, transportation, logistics.
  • Blockchain 4.0: Ekosistem business-friendly untuk "world of commons".
Skalabilitas — Masalah Utama
  • Masalah: Bitcoin hanya bisa proses ~7 transaksi/detik. Visa bisa 24,000/detik. Block size kecil + waktu mining 10 menit = bottleneck.
  • Solusi 1 — Storage Optimization: Delete transaksi lama yang tidak relevan agar ledger lebih ringan.
  • Solusi 2 — Bitcoin-NG: Pisahkan block jadi dua: Key block (pilih leader/miner) + Microblocks (transaksi saja). Throughput jauh lebih tinggi.

⚡ Poin Kunci untuk Ujian

  1. Blockchain = DLT paling populer. Terdesentralisasi, immutable, consensus-based.
  2. Setiap block berisi header (dengan parent hash) + body (transaksi). Genesis block = block pertama.
  3. Hash function = satu arah, avalanche effect. Merkle Tree = efisien untuk banyak transaksi.
  4. ECC > RSA di blockchain (256-bit ECC ≈ 3072-bit RSA dalam keamanan).
  5. 3 tipe: Public (terbuka) → Consortium (semi) → Private (tertutup).
  6. PoW = puzzle mining, aman tapi boros energi. PoS = jaminkan aset, lebih efisien.
  7. Evolusi: 1.0 (currency) → 2.0 (smart contracts) → 3.0 (segala sektor) → 4.0 (business ecosystem).
  8. Skalabilitas masalah utama blockchain. Bitcoin-NG = solusi dengan key block + microblocks.
